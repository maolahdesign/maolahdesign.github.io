Figma Automator 完整介紹教學

本文件整理 Figma Automator 外掛程式的核心概念、功能特性與實務用途，適合作為課程講義或內部教材使用。 資料整理時間：2026 年 7 月

一、什麼是 Figma Automator？

Automator 是由 Diagram（創辦人 Jordan Singer） 開發的 Figma 免費外掛程式，目前已有超過 8.7 萬名使用者、3,300+ 評分。它的核心概念很簡單：

把 Figma 裡「重複、繁瑣、耗時」的手動操作，變成一鍵可重播的自動化流程。

與其每次都手動點選、複製、對齊、命名，Automator 讓你用「拖拉組裝」的方式，把一連串動作串接成一條「自動化（Automation）」，之後只要按一下就能對選取的圖層重複執行。

教學重點： Automator 本身不是單一功能的工具，而是一個平台／容器——它把 Figma Plugin API 開放出來的 100 多種操作，包裝成可視覺化拖拉的積木，讓沒有寫程式背景的設計師也能組出自己的自動化流程。

二、核心功能與特性

1. 拖拉式流程建構（No-code Automation Builder）

  • 不需要寫程式，用滑鼠拖拉「動作積木（Action）」即可組成一條自動化流程

  • 動作可以依序排列，前一步的結果會傳遞給下一步

  • 建好的流程可以重複套用在不同的圖層／檔案上

2. 100+ 種內建動作（Actions）

Automator 把 Figma Plugin API 能做的事都包裝成現成的積木，常見類別包含：

  • 圖層操作：批次重新命名、對齊、排序、鎖定／解鎖、群組

  • 樣式與元件：套用顏色樣式、替換元件實例（Instance）、批次修改元件描述

  • 文字處理：批次修改文字內容、抓取／取代特定字串

  • 匯出相關：批次匯出圖層為圖片、依命名規則整理匯出檔名

  • 版面生成：依資料陣列自動生成網格版面、自動複製並調整尺寸產生多種變體（例如 Logo 各尺寸版本、Icon 集合）

3. 匯入／匯出 JSON，方便分享與重用

每一條自動化流程都可以匯出成 JSON 檔案，方便：

  • 團隊內部共享固定的操作流程（例如統一的命名規則、統一的匯出設定）

  • 直接匯入別人在社群分享的現成自動化，不必自己重新組裝

4. Automator Community（社群自動化庫）

Automator 有自己的社群網站 automator.community，使用者可以瀏覽、下載別人建立好的自動化流程，例如：

  • 依效果樣式（Effect Style）批次重新命名圖層

  • 匯出本地色彩樣式庫為色票卡片（含色票名稱與 HEX 色碼）

  • 將元件替換為實例並歸檔到指定頁面

教學重點： 對初學者來說，與其從零開始組裝，不如先去社群庫「站在巨人肩膀上」，抓現成的自動化來改，學習效率更高。

5. 支援 FigJam

Automator 同時支援 Figma Design 與 FigJam 兩種編輯器，代表無論是介面設計稿還是白板協作內容，都能套用自動化流程。

四、如何開始使用（教學步驟）

  1. 安裝外掛：在 Figma Community 搜尋「Automator」（開發者：Diagram），點選 Open in Figma 安裝

  2. 啟動外掛：在檔案內按右鍵 → Plugins → Automator，或使用 ⌘ /（Mac）／Ctrl /（Windows）快速指令搜尋

  3. 建立第一條自動化：

    • 點選「新增自動化（New Automation）」

    • 從左側動作清單拖拉需要的動作積木到流程區（例如：Rename → Align → Export）

    • 依序設定每個動作的參數

  4. 測試執行：先在少量圖層上選取並執行，確認結果符合預期

  5. 正式套用：選取要處理的完整圖層範圍，點擊執行，一次套用到所有選取物件

  6. 保存與分享：將流程匯出成 JSON，存放在團隊共用資源中，或上傳到 automator.community 分享給其他設計師

五、費用說明

Automator 是 完全免費 的 Figma 社群外掛程式，採用 Community Free Resource License 授權：

  • 沒有訂閱制、沒有額度限制

  • 所有 100+ 動作與匯入匯出功能皆免費開放使用

  • 社群自動化庫（automator.community）內容也可免費瀏覽與下載

教學提醒： 因為完全免費，Automator 很適合作為團隊導入「設計自動化」概念的第一步，之後若有更進階需求（例如跨平台批次生成、AI 輔助生成），可再評估 Figma Weave 或其他付費工具。

六、常見問題（FAQ）

Q1：Automator 跟 Figma 內建的「Rename layers with AI」有什麼不同？ Figma 內建的 AI 重新命名功能是「智慧判斷語意」自動取名；而 Automator 是「你自訂固定規則」的批次操作工具，兩者可以搭配使用：先用 AI 命名抓大方向，再用 Automator 套用團隊規範做精修。

Q2：不會寫程式也能用嗎？ 可以。Automator 的設計理念就是「拖拉組裝、不用寫程式」，這也是它最大的特色。

Q3：自動化流程可以跨檔案使用嗎？ 可以，透過匯出／匯入 JSON 的方式，同一條自動化流程能套用到不同檔案、甚至分享給其他團隊成員。

Q4：Automator 和「Page Automator」是同一個外掛嗎？ 不是。Page Automator 是另一款專注於「批次建立與管理頁面結構」的外掛；Automator（Diagram 出品）則是通用型、涵蓋 100+ 種圖層與版面操作的自動化平台，兩者可依需求分別安裝。
